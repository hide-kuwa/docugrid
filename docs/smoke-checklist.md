# 手動スモークチェック（リリース前・大きな変更後）

自動テスト（`backend/tests` の `pytest` + `frontend` の `npx tsc --noEmit`、GitHub Actions CI）に加え、ブラウザで次を確認する。

## 前提

- バックエンド: `http://127.0.0.1:8000`（`GET /health` が `{"status":"ok"}`）
- フロント: `http://localhost:3000`
- `frontend/.env.local` の `NEXT_PUBLIC_API_BASE` が `http://localhost:8000/api`（または同等）
- 大きな API 変更後は **バックエンドを再起動** してから確認する

## 自動チェック（コミット前）

```bash
# リポジトリルート
npm run test:backend
cd frontend && npx tsc --noEmit
```

## チェックリスト

1. **ログイン**  
   - `/login` でメール・パスワード・担当を選びログインできる。  
   - 誤パスワードで拒否される。

2. **PDF アップロード（ドロップで自動保存）**  
   - グリッドのスロットに PDF をドロップすると **自動保存** され、カードが「収納済み」になる。  
   - 版ラベル（例: `v1.0.0`）が表示される。  
   - ドロップ直後にビューアは **開かない**（プレビュー / 編集ボタンから開く）。

3. **プレビュー / 編集**  
   - 「プレビュー」で読み取り専用ビューアが開き、1 ページ目が表示される。  
   - 「編集する」で注釈・並べ替えが使える。

4. **注釈**  
   - マーカー / 枠 / 線 / チェックのいずれかを適用し、プレビューに反映される。  
   - 注釈後 **約 8 秒** で自動版スナップショット（minor）が作成され、版ラベルが進む（例: `v2.1.0`）。

5. **チェックモード（税理士向け）**  
   - 「編集する」→ 作業保存 / チェック開始 / 承認 / 差戻し が動く。  
   - 作業保存・チェック開始・承認後、グリッドの版ラベルとステータスバッジが更新される。  
   - 履歴パネルから旧版を選ぶと、サーバー上の **不変 PDF** が表示される。  
   - ビューアでページをめくると `page_view` 監査イベントが記録される（8 秒ごとにバッチ送信）。

6. **監査タイムライン（マトリクス）**  
   - MatrixGrid の「監査タイムライン」を開くと、直近の業務イベントが表示される。

7. **業務監査（設定画面）**  
   - 設定 → 操作履歴 → 顧客を選び「タイムライン読込」でイベント一覧が出る。  
   - CSV / JSON エクスポートができる（承認権限）。

8. **AI 設定**（設定 → 外部連携）  
   - OpenAI / Gemini キーを保存でき、GET ではキー本体が返らない（`設定済み` 表示のみ）。

9. **自動振り分け**（該当 UI を使う場合）  
   - 一括ドロップゾーンに PDF を入れると分類候補が表示され、確認後にスロットへ収納される。

10. **不足資料 / 今日やること**  
   - 期のヘッダーまたはダッシュボードに不足・承認待ち・差戻しが表示される。

11. **リロード復元**  
   - ページをリロードしても、保存済みスロットの PDF と版ラベルが復元される。

12. **DocuGrid 同期（ページ並び）**  
   - PDF をスロットに保存後、「ページの並び」パネルで DnD すると **セッション同期** バッジが dirty → 約 2 秒後に saved になる。  
   - 「今すぐ保存」で手動同期できる。  
   - リロード後に同じスロットを開くと、保存済みのページ順が復元される。

13. **再アップロード時の処理中表示**  
   - 既存スロットに PDF を差し替えると、一時的に「処理中」バッジが出て、完了後に版ラベルが更新される。

14. **エラー時**  
   - バックエンドを止めた状態で操作し、フロントが異常終了せずメッセージまたは接続エラー表示になる。

---

失敗した項目は Issue にメモし、再現手順とブラウザを添える。
