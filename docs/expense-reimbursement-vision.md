# 営業経費精算ビジョン

最終更新: 2026-06-17

## 位置づけ

営業経費精算を TAXX / DocuGrid の **キャプチャファースト + 自動監査 + トーンダウン** 思想で再設計する。営業は「パシャリ」だけ、経理は **赤カードだけ** を見る。

UI 基盤: [`capture-gallery-ux-vision.md`](capture-gallery-ux-vision.md)  
エコシステム: [`taxx-ecosystem-development-plan.md`](taxx-ecosystem-development-plan.md)

---

## 1. ペインの構造

| 側 | ボトルネック |
|----|--------------|
| **営業** | 入力の手間・忘却（交際費の相手先、交通費の経路）、財布の領収書紛失・デブ化 |
| **経理** | インボイス確認、定期控除、交際費 1 人当たり基準、重複・二重申請の検知 |

---

## 2. 目標体験

### 2.1 ログイン即カメラ & Pinterest ギャラリー

- タクシー・カフェ退出直後にアプリ → 1 秒でカメラ → 撮影完了
- レシートが Masonry カードとして敷き詰められる
- **電子帳簿保存法**（スキャナ保存要件: タイムスタンプ・解像度）をバックグラウンドで満たし、紙はその場で破棄可能に（要件は法改正に追随する KB で管理）

### 2.2 コンテキスト RAG（誰と・何のために）

| 入力 | AI の推論 |
|------|-----------|
| OCR: 店名・日時・金額 | Google カレンダー / CRM とマッチング |
| サジェスト UI | 「〇〇商事 佐藤様ほか 1 名との会食ですか？ [はい]」→ 1 タップで相手先タグ完了 |
| インボイス | 店名から登録番号検索 → 国税庁公表サイトとリアルタイム突合 |
| 交際費基準 | 5,000 円 / 1 万円基準・1 人当たり金額を裏で計算 |

### 2.3 領収書のない交通費の自動カード化

- GPS / カレンダー移動履歴から経路・運賃を算出
- 領収書なしでも **自動生成交通費カード** がギャラリーに並列表示
- 従業員マスタの **定期券区間** を経路 API の除外条件に強制マッピング → 重複申請をロジックで遮断
- 駅すぱあと等の経路検索 API 連携（デジタル庁・MaaS データ標準化の動向を参照）

### 2.4 経理の月次締め

- 合格カードはトーンダウンして背景に溶ける
- 経理が見るのは **赤ハイライト + 最上部ピン留め** の数件のみ
- 仕訳生成は自動監査パス後にバッチ

## 3. 監査ロジック（CHECK）

| 不正・ミス | 検知 |
|------------|------|
| 同一領収書の使い回し | 店名・日時・金額 + **画像ハッシュ** の重複 → 即赤ピン |
| 定期券区間内の交通費 | マスタ定期区間 × 経路 API 除外 |
| 失効インボイス | 公表サイト突合 |
| 交際費基準超過 | 人数 × 単価の自動計算 |

グレー判定は [`taxx-ecosystem-development-plan.md`](taxx-ecosystem-development-plan.md) §3 と同様、ワンタップ確認。

## 4. 法規・コンプライアンス

| 法規 | 適用 |
|------|------|
| 電子帳簿保存法 | スキャナ保存要件（解像度・タイムスタンプ・検索性） |
| インボイス制度 | 適格請求書・登録番号確認義務 |
| 法人税法 | 交際費等の損金不算入（しおり） |
| 内部統制 | 自動監査ログの追記専用記録（監査イベントと統合） |

---

## 5. プロトタイプ提案（最初の一歩）

**交際費のインボイス × カレンダー自動突合** から攻める。

| 理由 | 説明 |
|------|------|
| 入力ミス最大 | 相手先・人数・会議費 vs 交際費の誤りが多い |
| 経理負荷最大 | 1 件あたりの突合時間が長い |
| 技術的にデモ可能 | OCR + カレンダー API + インボイス API で価値を早期証明 |

スコープ:

1. レシート撮影 → Masonry カード  
2. 店名・日時 OCR → カレンダー候補サジェスト  
3. インボイス番号突合 → 緑 / 赤  
4. 経理画面: 赤のみピン留め

## 6. TAXX エコシステム内の位置

```
営業経費カード ──► 仕訳（経費科目）
       │
       ├─► 支払調書（源泉・不動産使用料等）
       │
       └─► インボイス・電帳法監査ログ
```

給与・源泉側との接点: 外注先個人への支払 → 源泉 2 軸マッピング（`taxx-ecosystem-development-plan.md` §3）
