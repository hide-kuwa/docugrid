# キャプチャファースト・Pinterest 風ギャラリー UX

最終更新: 2026-06-17

## 位置づけ

ログイン直後に **カメラが即起動** し、撮影した資料が **Pinterest 風 Masonry** で視覚的に並ぶ UI。既存の退屈な「文字だらけリスト型」会計ソフトの代替となる **キラー UX**。

適用範囲:

- 税理士事務所の資料回収（DocuGrid マトリクス補完）
- 従業員のまるふ・控除証明書提出（→ [`payroll-withholding-year-end-vision.md`](payroll-withholding-year-end-vision.md)）
- 営業経費精算（→ [`expense-reimbursement-vision.md`](expense-reimbursement-vision.md)）

親文書: [`taxx-ecosystem-development-plan.md`](taxx-ecosystem-development-plan.md) §2.3 トーンダウン、§2.4 チェックリスト同期

---

## 1. UX 原則

| 原則 | 説明 |
|------|------|
| **Zero-Click キャプチャ** | ログイン → カメラ or 巨大 D&D エリア。IT 苦手な社長でも迷わない |
| **視覚優位** | ファイル名・日付リストではなくサムネイルで「あの緑の領収書」「あの手書きマル扶」を瞬時識別 |
| **監査と同期** | OCR/自動監査の結果をカードの枠・不透明度・ピン留めで表現 |
| **手数最小化** | デジタル庁が掲げる操作手数削減の極致形 |

---

## 2. 画面フロー

```
ログイン
  └─► キャプチャホーム（カメラ / D&D が主役）
        └─► シャッター or ドロップ
              └─► 非同期アップロード + OCR キュー
                    └─► Masonry ギャラリーにカード追加
```

### 2.1 キャプチャホーム

- フルスクリーンに近いカメラプレビュー or アップロードゾーン
- マトリクスへの振り分けは **裏側**（分類 API / 要確認キューは既存 `page.tsx` パターンを拡張）

### 2.2 Masonry ギャラリー

- レシート（縦長）・通帳（横長）・申告書（A4）など **高さバラバラ** を隙間なく敷き詰め
- 技術: Tailwind `columns-*`、または `react-masonry-css` / CSS Grid masonry（ブラウザ対応に留意）
- `next/image` による WebP 化・リサイズ + **Lazy Loading**（高解像度写真の性能対策）

### 2.3 カードステータスとトーンダウン連動

| ステータス | 見た目 | 挙動 |
|------------|--------|------|
| 解析中 | 枠グレー・スピナー | — |
| 整合性 OK | 枠が淡い緑 → **トーンダウン**（背景に溶け込む） | チェックリスト進捗と連動 |
| 要確認 | 枠ビビッド赤・**画面上部にピン留め** | スポットライト |
| 完了（人間確認済） | スワイプ or チェックでフェードアウト | 次の要確認へ |

ゲーミフィケーション: 確認完了でカードが消え、チェックリストが進む → 監査作業の心理的負荷低減。

## 3. 既存 DocuGrid との関係

| 既存 | ギャラリー UX |
|------|----------------|
| 顧問先 × 期間 × **スロット** マトリクス | 期間横断の **視覚的探索**・モバイル回収入口 |
| 要確認キュー（分類） | 赤ピンカード = 要確認キューのギャラリー表現 |
| PDF ビューア | カードタップで既存ビューアを開く |
| DATA 正規化ハブ | OCR 結果 → `client-master`（`client-data-vision.md`） |

**置き換えではなく補完:** 実務家はマトリクスで期間管理、ギャラリーで「今日撮ったもの」「要確認だけ」を処理。

## 4. 技術メモ（検証済み前提）

| 項目 | 方針 |
|------|------|
| スタック | React / TypeScript / Next.js / Tailwind（現行 frontend と一致） |
| 画像 | `next/image`、Firebase Storage 等（インフラは `architecture.md` と整合） |
| ソート | 日付順だけでなく **要確認ステータス優先** + トップピンで確認漏れ防止 |
| Masonry の視線 | 左→右時系列は崩れる → 監査フェーズはステータスソートを優先 |

### CHECK

| 懸念 | 対策 |
|------|------|
| 大量高解像度画像 | Lazy load + サムネイル層 |
| 時系列の見づらさ | フィルタ: 今日 / 要確認 / 未分類 |
| アクセシビリティ | カメラ以外にファイル選択・音声フィードバック（将来） |
